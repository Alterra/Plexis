Defined Constants

Below is a list of defined constants that Plexis uses, along with a description of what each constants means, and its value definition.

CMS_VERSION

This constant contains the cms release version string.

CMS_BUILD

This constant contains the cms build (aka revision) of the cms as an integer.

REQ_DB_VERSION

This constant contains the required database version for the cms to run properly as a string.

ROOT

This constant contains the full drive path to your plexis installation folder. No trailing directory separator included.

DS

This constant is just an alias of the PHP constant 'DIRECTORY_SEPARATOR', but obviously much shorter.

SYSTEM_PATH

This constant contains the full path to the "{Plexis Install Dir}/system" folder. No trailing directory separator included.

SITE_URL

This constant contains the current URL to the website as a string. Take note, that it will also contain "/?url=" IF Mod Rewrite is not enabled by apache. This constant is best used for template (a href='') links.

BASE_URL

This constant contains the current URL to the website as a string. The difference is, BASE_URL will not include the "/?url=" when Mod Rewrite is disabled.



Global Variables

Below is a list of global variables (stored in the $GLOBALS variable) that plexis defines, As well as when they are defined.

controller

This global variable contains the controller / module name to be loaded. This global becomes available just before the 'pre_controller' event is fired, and the controller is constructed.

action

This global variable contains the method name to be called in the controller. This global becomes available just before the 'pre_controller' event is fired, and the controller is constructed.

is_module

This global variable contains a bool, defining whether we are loading a module or not. This global becomes available just before the 'pre_controller' event is fired, and the controller is constructed.

language

This global variable contains the selected language for this user (including guests), based off of the language cookie, or browsers selected language. This global becomes available just after the core controller is finished constructing.
