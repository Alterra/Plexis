Events

Plexis uses an Event Handler, which allows callbacks to made once a particular action is made. This allows your plugins and modules to easily hook into the system, and process actions at the optimal time. For more information on loading the class, and registering callbacks, please check out This Page. Below is a list of events in Plexis CMS that your custom code can register to receieve a callback from. Callback Recieved Arguments is a list of arguments that are passed to your callback method when the event is triggered.

Controller Initialized

This event is triggered once the application controller has finished constructing. To register for this event, use the following code:

$this->Events->register('controller_init', '$callback');
Callback Recieved Arguments:

None

User logging In

This event is triggered when a user logs into his account. To register for this event, use the following code:

$this->Events->register('user_logged_in', '$callback');
Callback Recieved Arguments:

  1. [Type: Int]: ID of the account.
  2. [Type: String]: Username of the account

User logging Out

This event is triggered when a user logs out of his account. To register for this event, use the following code:

$this->Events->register('user_logged_out', '$callback');
Callback Recieved Arguments:

  1. [Type: Int]: ID of the account.
  2. [Type: String]: Username of the account

New Account Registration

This event is triggered when a new account is registered. To register for this event, use the following code:

$this->Events->register('account_created', '$callback');
Callback Recieved Arguments:

  1. [Type: Int]: ID of the account.
  2. [Type: String]: Username of the account
  3. [Type: String]: The account password
  4. [Type: String]: Email Address to the account
  5. [Type: String]: Users current IP address

Account Password Change

This event is triggered when a user or admin changes the password of an account. To register for this event, use the following code:

$this->Events->register('password_change', '$callback');
Callback Recieved Arguments:

  1. [Type: Int]: ID of the account.
  2. [Type: String]: New password to the account

Account Email Change

This event is triggered when a user or admin changes the email address of an account. To register for this event, use the following code:

$this->Events->register('email_change', '$callback');
Callback Recieved Arguments:

  1. [Type: Int]: ID of the account.
  2. [Type: String]: Old account email address
  3. [Type: String]: New account email address

Vote Submission

This event is triggered when a user votes. To register for this event, use the following code:

$this->Events->register('user_voted', '$callback');
Callback Recieved Arguments:

  1. [Type: Int]: ID of the account.
  2. [Type: Int]: The votesite ID
