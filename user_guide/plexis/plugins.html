Creating Plugins

Plexis comes equiped with a plugin system, that act just like hooks in any other framework / cms. Plugins are mainly just normal classes, but their method's (depending on name) are called based on where the script is at that current point. Plugins are perfect for registering Events and callbacks ;)

Plugin Layout

Below is the base of any plugin. The methods listed here are Required in order for your plugin to work correctly when loaded.

namespace Plugins;

class Plugin_name
{
	public function pre_system()
	{
		// This code is executed very early in the cms, before any database connections are made,
		// right after the URL is routed.
	}
	
	public function pre_controller() 
	{
		// This code is executed right before the controller is constructed. Database connections are 
		// established right before this point.
	}
	
	public function post_controller_constructor() 
	{
		// This code is executed imediatly after the controller is finsihed constructing, but before
		// the page method is loaded.
	}
	
	public function post_controller() 
	{
		// This code is executed after the template has rendered the template, and pushed its contents
		// to the browser.
	}
}

All plugin file names must have the first letter capitalized! The Classname and Namespace are also Capitalized.
