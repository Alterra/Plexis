List of Plexis Functions

Bwlow is a full list of functions that plexis defines. Each function will have a description of what it does, all parameters explained, and the return values.

path()

The path() function is used to create paths to file and folders, using the correct directory separator for the operating system. Putting 3 parameters ('test', 'folder', 'file') would return a string like so "test/folder/file". It is important to use this function because it will put the correct operating system specific directory separator (/ or \) between the paths given. This function will also check for illegal characters in filepaths, and show an error if there is any found.

path( array of $params);
Parameters:

  1. $params [Type: String | Array]: A folder or filename. Can also be passed as an array of folders and/or a filename as well.

Return Values:
(String) Returns the parameters as a filepath, with the correct operating system directory separator.

load_class()

This function is used to load and store core classes statically that need to be loaded for use. If loading the same class twice with this function, it will return the same instance as before. This function should only be used to load classes from the 'system/core' folder. Library classes should be loaded via the Loader class.

load_class($className, $type, $supress);
Parameters:

  1. $className [Type: String]: The class name we are trying to load.
  2. $type [Type: String, Default: 'core']: The type of class (library, core, etc etc)
  3. $supress [Type: Bool, Default: false]: Supress error on failed initialization. Setting to true, will not show an error screen if the class fails to start (IE: an exception was thrown in the constructor)

Return Values:
(Object | false) Returns the class object, or false if the class failed to initialize

show_error()

This function sends an error to be logged, and / or displays an error message on the screen, depending on the cms enviorment (production, or development).

show_error($message, $args, $level, $fatal);
Parameters:

  1. $message [Type: String]: This would be the array key in the 'messages.php' language file, or custom error text.
  2. $args [Type: Array, Default: Null]: An array of arguments to replace the "%s"'s in the error message. Set to false or null to skip.
  3. $level [Type: Constant, Default: E_ERROR]: The error level of this error. The constant passed is the same as any php error constant, Ex: E_ERROR shows an error, and E_WARNING shows a warning message. The level of the error determines IF an error is displayed to the user, and whether the script stops and quits the rest of the page load (Depending on the enviroment). Errors can be silenced using the debug "silent_mode()" method.
  4. $fatal [Type: Bool, Default: false]: If set to true, then "silent_mode()" will automatically be set false, and will display the error indefinatly.

Return Values:
(None).

show_404()

This function stops script execution, and displays the 404 page.

show_404();
Parameters:

(None)

Return Values:
(None)

log_message()

This function logs a system message into the system_logs.log file. These message can be of 2 type's, error or info. Error message are for logging none fetal, class error's, and info messages to be logged for any reason, such as why a class method is preforming an action a certain way. Each time this function is called, it adds a new line to the log file.

log_message($type, $message);
Parameters:

  1. $type [Type: String]: The message type (info | error)
  2. $message [Type: String]: The message to be logged in the log file

Return Values:
(Bool) True on success, false if writting to logfile failed

get_instance()

This function returns the current loaded controller instance. All class variables and methods will be publically accessable.

$Controller = get_instance();
Parameters:

(None)

Return Values:
(Object) Returns the controller class instance

register_instance()

This function sets the instance of the controller, to the specified class object. Future calls to the 'get_instance()' function will return the object given here

register_instance($Object);
Parameters:

  1. $Object [Type: Object]: The object we are setting as the global instance of the controller

Return Values:
(None)
